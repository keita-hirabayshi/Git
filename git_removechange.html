ファイルの変更を取り消す git checkout --(ファイル名) checkout によりステージ上の指定されたファイルを読み込み、ワークツリーに反映させることで変更を取り消す git checkout --(ディレクトリ名) git checkout --. * --をつけているのは、ブランチ名とファイル名が被ったときに、どちらを指しているのかGitがわからなくなるのを避けるため ステージに追加した変更を元に戻したいとき git reset HEAD(ファイル名) HEADは今自分がいるブランチにおける最新のコミットのことで、reset HEADでリセットしてあげるという意味になる 仕組みとしては、レポジトリに記憶してある一番最近のコミットを呼び出し、ステージに上書きしてあげる git reset HEAD(ディレクトリ名) git reset HEAD. (前変更を取り消す場合) ＊指定した変更をステージから取り除くだけで、ワークツリーの変更は取り消せない 直前のコミットをやり直す git commit -amend 仕組みはワークツリーからgit addにて直前に変更した内容をstageから上書きしてコミットすることでコミットをやり直す *リモートレポジトリにプッシュしたコミットはやり直し禁止
