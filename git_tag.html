タグ一覧を表示する git tag コミットを参照するためにわかりやすい名前をつけるのがタグ(リリースポイントに使うこと多い) バグ発生時の修正が素早くできる git tag -l"202205" タグを作成する ２パターンある(注釈付きと軽量版) 注釈付き (git tag -a(タグ名) -m"メッセージ") git tag -a 20220531_01 -m"version 20220531_01" *コメントや作成した人の署名も追加できる 軽量版 git tag(タグ名) git tag 20220531_01 # 後からタグ付けする　　git tag (タグ名) (コミット名) *タグ名はつけられるが、メッセージ等は追加できない。 タグのデータを表示する git show (タグ名) →タグのデータと関連づけられたコミットを表示する タグをつけると、エラー発生時にスムーズに対応できるので、リリースポイントといった重要な時には記録するようにする。 タグをリモートレポジトリに送信する git push (リモート名) (タグ名) # 一斉に送信する場合 git push origin --tags (tagsとすることで、リモートにはないローカルのタグを一斉に送信する)
